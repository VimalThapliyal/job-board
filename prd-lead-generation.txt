## üìä System Requirements

### **Functional Requirements:**

#### **1. Lead Capture System**
- **Job Detail Pages:** Replace "Apply Now" with application forms
- **Form Fields:** Name, email, phone, experience, resume, cover letter
- **File Upload:** PDF, DOC, DOCX (max 5MB)
- **Validation:** Required fields, email format, file type
- **Success Flow:** Confirmation page, email to candidate

#### **2. Lead Management Dashboard**
- **Admin Panel:** View all leads, filter by status
- **Lead Status:** New, Qualified, Contacted, Sold, Rejected
- **Search & Filter:** By job, company, date, experience
- **Export:** CSV for company outreach

#### **3. Company Outreach System**
- **Automated Emails:** Send leads to hiring companies
- **Lead Qualification:** Experience match, skills alignment
- **Pricing Tiers:** Based on job level and candidate quality
- **Follow-up:** Automated reminders for unpaid leads

#### **4. Payment Processing**
- **Stripe Integration:** Handle lead payments
- **Invoice Generation:** Automatic billing for companies
- **Payment Tracking:** Revenue analytics and reporting

### **Non-Functional Requirements:**

#### **Performance:**
- **Page Load:** < 2 seconds
- **Form Submission:** < 3 seconds
- **File Upload:** < 10 seconds for 5MB files
- **Concurrent Users:** 100+ simultaneous applications

#### **Security:**
- **Data Encryption:** All personal data encrypted
- **File Security:** Secure resume storage
- **GDPR Compliance:** Data protection and privacy
- **Access Control:** Admin-only lead management

#### **Scalability:**
- **Database:** MongoDB Atlas (auto-scaling)
- **File Storage:** Vercel Blob (unlimited)
- **Email:** Resend (10,000 emails/month)
- **CDN:** Vercel Edge Network

## üóÑÔ∏è Database Schema

### **Leads Collection:**
```typescript
interface Lead {
  id: string;
  jobId: string;
  jobTitle: string;
  company: string;
  
  // Candidate Info
  name: string;
  email: string;
  phone?: string;
  experience: string;
  skills: string[];
  
  // Application
  resumeUrl: string;
  coverLetter?: string;
  
  // Status
  status: 'new' | 'qualified' | 'contacted' | 'sold' | 'rejected';
  qualificationScore: number; // 0-100
  
  // Metadata
  createdAt: Date;
  updatedAt: Date;
  ipAddress: string;
  userAgent: string;
}
```

### **Companies Collection:**
```typescript
interface Company {
  id: string;
  name: string;
  email: string;
  phone?: string;
  
  // Billing
  stripeCustomerId?: string;
  leadPricing: {
    junior: number; // $50
    mid: number;    // $100
    senior: number; // $150
    lead: number;   // $200
  };
  
  // Settings
  autoPurchase: boolean;
  maxBudget: number;
  
  createdAt: Date;
  updatedAt: Date;
}
```

### **Transactions Collection:**
```typescript
interface Transaction {
  id: string;
  leadId: string;
  companyId: string;
  amount: number;
  currency: string;
  status: 'pending' | 'completed' | 'failed';
  stripePaymentId?: string;
  
  createdAt: Date;
  completedAt?: Date;
}
```

## üîÑ User Flows

### **Candidate Application Flow:**
1. **Browse Jobs:** User finds job on job board
2. **View Details:** Clicks on job to see full description
3. **Apply:** Fills out application form
4. **Upload Resume:** Attaches resume file
5. **Submit:** Form validation and submission
6. **Confirmation:** Success page and email confirmation

### **Lead Processing Flow:**
1. **New Lead:** Application submitted to database
2. **Qualification:** Automated scoring based on experience/skills
3. **Company Match:** Find relevant companies for the job
4. **Outreach:** Send qualified leads to companies
5. **Purchase:** Company buys lead
6. **Payment:** Process payment via Stripe
7. **Handoff:** Provide candidate contact to company

### **Admin Management Flow:**
1. **Dashboard:** View all leads and analytics
2. **Qualification:** Manually review and score leads
3. **Outreach:** Send leads to companies
4. **Payment Tracking:** Monitor revenue and payments
5. **Reporting:** Generate revenue and performance reports
